# Benchmark Baseline - 2026-05

Baseline benchmark results for tracking performance regressions.

## Environment

- **OS:** darwin (macOS)
- **Arch:** arm64
- **CPU:** Apple M4
- **Go version:** 1.22
- **Date:** 2026-05-13

## Policy

- Run benchmarks before each release
- Regression >20% = PR blocker
- Update this file with new baselines after optimizations
- Name file: `YYYY-MM.md`

## How to Run

```bash
# All benchmarks
make bench

# Specific package
go test -bench=. -benchmem -run=^$ ./internal/domain/valueobject

# With memory profiling
go test -bench=BenchmarkMoney -benchmem -memprofile=mem.out -run=^$ ./internal/domain/valueobject

# Bcrypt calibration (few iterations due to slowness)
go test -bench=BenchmarkBcryptCost -benchtime=5x -run=^$ ./internal/infrastructure/auth
```

## How to Update Baselines

After optimization work:

```bash
# Run and save to monthly file
go test -bench=. -benchmem -run=^$ ./... | tee docs/benchmarks/$(date +%Y-%m).md

# Compare against previous baseline
benchstat docs/benchmarks/2026-05.md docs/benchmarks/2026-06.md
```

Install benchstat:
```bash
go install golang.org/x/perf/cmd/benchstat@latest
```

## Regression Detection

CI job should run benchmarks on PR and compare against main branch:

```yaml
# .github/workflows/bench.yml (future)
- name: Run benchmarks
  run: go test -bench=. -benchmem -run=^$ ./... > bench.txt

- name: Compare against main
  run: benchstat main-bench.txt bench.txt > bench-diff.txt

- name: Fail on regression >20%
  run: grep -E "\+[2-9][0-9]%|\+[0-9]{3}%" bench-diff.txt && exit 1 || exit 0
```
